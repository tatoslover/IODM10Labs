🐳 Exercise 1: Docker Implementation

▼
Skills Demonstrated: Dockerfile creation, image optimization, containerization best practices, and Docker Hub publishing

Challenge: Create a DockerFile for a Node.js application without database dependency and publish to Docker Hub for public access.

📄 Dockerfile Implementation
FROM node:19-alpine WORKDIR /app COPY . . EXPOSE 3000 RUN npm install CMD ["npm", "start"]
🔧 Docker Commands
# Build the Docker image docker build -t socket-chat . # Run the container docker run -p 3000:3000 socket-chat # Tag for Docker Hub docker tag socket-chat username/socket-chat:latest # Push to Docker Hub docker push username/socket-chat:latest # Pull and run from Docker Hub docker pull username/socket-chat:latest docker run -p 3000:3000 username/socket-chat:latest

🎯 Key Implementation Features

  • Alpine Linux Base: Minimal image size for efficiency
  • Working Directory: Organized file structure in container
  • Port Exposure: Proper network configuration
  • Dependency Installation: Automated npm install process
  • Start Command: Production-ready application launch

⚙️ Exercise 2: GitHub Actions CI/CD

▼
Skills Demonstrated: GitHub Actions workflow creation, automated testing, Docker Hub integration, and continuous deployment

Challenge: Create GitHub Actions workflow that automatically builds and pushes Docker images to Docker Hub when commits are made to the main branch.

🔄 GitHub Actions Workflow (.github/workflows/cicd.yml)
name: CI/CD on: push: branches: [main] jobs: build: runs-on: ubuntu-latest strategy: matrix: node-version: [19.x] steps: - name: Checkout repository uses: actions/checkout@v3 - name: Set up Node.js ${{ matrix.node-version }} uses: actions/setup-node@v3 env: PUPPETEER_SKIP_CHROMIUM_DOWNLOAD: "true" with: node-version: ${{ matrix.node-version }} - name: Install dependencies run: npm install - name: Login to Docker Hub uses: docker/login-action@v2 with: username: ${{ secrets.DOCKER_HUB_USERNAME }} password: ${{ secrets.DOCKER_HUB_ACCESS_TOKEN }} - name: Set up Docker Buildx id: buildx uses: docker/setup-buildx-action@v2 - name: Build and push id: docker_build uses: docker/build-push-action@v4 with: context: ./ file: ./Dockerfile push: true tags: ${{ secrets.DOCKER_HUB_USERNAME }}/mvc-structure:latest - name: Image digest run: echo ${{ steps.docker_build.outputs.digest }}

🎯 CI/CD Pipeline Features

  • Automated Triggers: Builds on every main branch push
  • Multi-Node Testing: Matrix strategy for Node.js versions
  • Dependency Management: Automated npm installation
  • Secure Authentication: GitHub Secrets for Docker Hub
  • Build Optimization: Docker Buildx for advanced features
  • Automated Publishing: Direct push to Docker Hub registry
